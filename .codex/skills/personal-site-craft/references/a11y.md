# Accessibility

## Visual Effects

Canvas/WebGL/shader effects must be decorative unless they carry content.

- Mark decorative canvas layers with `aria-hidden="true"` and `role="presentation"`.
- Respect `prefers-reduced-motion`.
- Disable or simplify effects in forced-colors/high-contrast contexts.
- Expose controls for intensity/scanlines/font size when effects can affect readability.
- Keep CPU/GPU cost low; pause heavy animation when hidden or when intensity is zero.

## Themes And Contrast

If a theme or readable palette exists, apply it consistently to:

- page text;
- chips;
- borders;
- terminal overlay;
- buttons;
- links;
- shader/background colors.

Do not create separate overlapping "themes" and "a11y color modes" unless the distinction is real. If the user says one button, collapse controls into one panel.

Check that dim labels remain readable across palettes. Avoid relying on hue alone.

## Terminal UI

Terminal overlays should be keyboard usable:

- focus input on open;
- keep focus visible after Tab;
- close with Escape;
- provide accessible names for close buttons and inputs;
- avoid trapping users without an exit.

If commands are fake, make them internally consistent: files should behave like files, binaries like binaries, `cat`, `ls`, `cd`, `rm`, `sudo`, and tab completion should follow familiar conventions.

## Retro UI

Retro UI can use loud colors and old-school layout, but must still:

- avoid mobile horizontal overflow unless intentionally scrollable;
- provide labels for form controls;
- use button text that fits on mobile;
- keep close controls visible and non-destructive-looking;
- avoid red hover for ordinary close buttons.

## Verification

After relevant changes, verify:

- desktop and mobile width;
- no incoherent overlap;
- no horizontal overflow on mobile;
- readable text in all themes;
- visible focus states;
- reduced-motion behavior if animation is involved.
